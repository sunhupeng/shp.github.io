一、选择题（1-20）
1计算机硬件能直接执行的只能是（ ）。
A.符号语言   B.机器语言   C.汇编语言   D.高级语言
B. 机器语言
解题思路CPU 只能识别和执行二进制机器码（0/1序列）。汇编语言需经汇编器转换为机器码，高级语言需编译/解释，符号语言不是标准术语。
2运算器的核心部件是（ ）。
A.数据总线   B.数据选择器   C.累加寄存器   D.算术逻辑运算部件
D. 算术逻辑运算部件（ALU）
解题思路运算器 = ALU + 寄存器 + 数据通路，核心是完成算术/逻辑运算的 ALU。数据总线是传输通道，累加器是暂存器。
3下列数中最大的数是（ ）。
A. (1000101)2   B. (01101000)BCD   C. (65)8   D. (61)16
D. (61)16 = 97
解题思路：全部转为十进制比较 A. (1000101)2 = 1×26+0+0+0+1×22+0+1 = 64+4+1 = 69 B. (01101000)BCD：BCD 每4位表示一位十进制数 → 0110=6, 1000=8 → 68 C. (65)8 = 6×81+5 = 48+5 = 53 D. (61)16 = 6×16+1 = 96+1 = 97 ← 最大
4在一个 8 位二进制数的机器中，补码表示数的范围从（ ）到 +127。
A.-256   B.-128   C.-127   D.-255
B. -128
解题思路n位补码表示范围：-2n-1 ~ +2n-1-1
8位补码：-27 ~ +27-1 = -128 ~ +127
10000000（-128）到 01111111（+127）
5在定点二进制运算器中，减法运算一般通过（ ）来实现。
A.原码运算的二进制减法器   B.补码运算的二进制加法器
C.原码运算的十进制加法器   D.补码运算的二进制减法器
B. 补码运算的二进制加法器
解题思路a - b = a + (-b)。将减数取补码后与被减数相加，即用加法器实现减法。统一用补码使加减法共用同一套电路。
6变形补码加法，双符号位为（ ）时表示发生了正溢出。
A.00   B.01   C.10   D.11
B. 01
解题思路变形补码（双符号位）判断溢出规则：
00 = 正数，无溢出   11 = 负数，无溢出
01 = 正溢出（两个正数相加，结果超出正数范围）
10 = 负溢出（两个负数相加，结果超出负数范围）
7将 -33 以补码形式存入 16 位寄存器中，则寄存器内容为（ ）。
A.00DFH   B.DDDFH   C.FFDFH   D.00FDH
C. FFDFH
解题思路：负数的补码 = 正数取反 + 1 ① 33 的二进制（16位）= 0000 0000 0010 0001 = 0021H ② 取反 → 1111 1111 1101 1110 = FFDEH ③ +1 → 1111 1111 1101 1111 = FFDFH
速算技巧
从低位开始找第一个1，之前的位保持不变，之后（高位）全部取反。33=...0010 0001，第一个1在第0位，高位取反得...1101 1111 = FFDFH
8只能发现奇数个错误，无纠错能力的校验码是（ ）。
A.CRC 码   B.海明码   C.奇偶校验码   D.以上都不是
C. 奇偶校验码
解题思路 - 奇偶校验码：添加1位校验位使1的总数为奇/偶，只能检测奇数个错误，无法纠错 - CRC码：循环冗余校验，可检测多位错误，常用于网络传输 - 海明码：可检测并纠正一位错误
9存储器主要用来存放（ ）。
A.程序   B.数据   C.微程序   D.程序和数据
D. 程序和数据
解题思路存储器存储两类信息：程序（指令序列）和数据（操作数/中间结果）。微程序是控制器中的概念，不是存储器的主要存储对象。
10双端口存储器能进行高速读/写操作，是因为采用了（ ）。
A.高速芯片   B.新型器件   C.两套相互独立的读写电路   D.流水技术
C. 两套相互独立的读写电路
解题思路双端口 RAM 有两套独立的地址线、数据线和读写控制电路，可同时访问不同存储单元，实现并行读写。跟芯片速度无关，是结构设计的改进。
11下列存储器中，不能与 CPU 直接交换信息的是（ ）。
A.只读存储器   B.随机存储器   C.缓存   D.辅助存储器
D. 辅助存储器
解题思路CPU 只能通过地址总线直接访问主存（RAM/ROM）和 Cache。辅助存储器（磁盘、U盘等）通过 I/O 接口与 CPU 交换数据，不能直接寻址。
内存层次：CPU → Cache → 主存(RAM/ROM) → 辅存(磁盘)
12某 SRAM 芯片，容量为 16K×8 位，则其地址线有（ ）根。
A.14   B.16K   C.16   D.8
A. 14
解题思路16K = 16 × 1024 = 16384 = 214
地址线根数 n 满足 2n = 存储单元数。
214 = 16384 = 16K，所以 n = 14 根地址线。
×8 位表示每个存储单元 8 位（1字节），数据线 8 根。
13采用指令 Cache 与数据 Cache 分离的主要目的是（ ）。
A.降低 Cache 的损失   B.提高 Cache 的命中率
C.降低 CPU 平均访存时间   D.减少指令流水线资源冲突
D. 减少指令流水线资源冲突
解题思路现代 CPU 使用流水线技术，取指和访存可能同时发生。若 Cache 合一，则取指令和读数据争抢同一端口 → 结构冲突。
分离后指令 Cache 和 数据 Cache 各有独立端口，可并行访问，减少资源冲突。
14以下寻址方式中，速度最快的是（ ）。
A.立即数寻址   B.寄存器间接寻址   C.直接寻址   D.寄存器寻址
A. 立即数寻址方式
解题思路：访问次数越少速度越快 A. 立即数寻址：操作数就在指令中，0次访存 → 最快
B. 寄存器寻址：操作数在寄存器，0次访存（但需先读取寄存器编号，略慢于立即）
C. 寄存器间接寻址：先读寄存器得到地址，再访存 → 1次访存
D. 直接寻址：地址在指令中，访存取操作数 → 1次访存
15单地址指令中为了完成两个数的算术运算，除地址码指明的一个操作数外，另一个常需采用（ ）。
A.堆栈寻址   B.立即寻址   C.隐含寻址   D.间接寻址
C. 隐含寻址方式
解题思路单地址指令只有一个地址码字段。两个操作数怎么来？
一个由地址码指出，另一个默认在累加器（ACC）中——这就是隐含寻址。
运算结果也存回累加器。早期 CPU 常用此方式，如指令 ADD X 表示 ACC = ACC + M[X]。
16用 2K×4 位的 RAM 芯片，设计 16K×8 位的 RAM 存储器，以下说法正确的是（ ）。
A.16片, 字扩展8, 位扩展2   B.16片, 字扩展4, 位扩展4
C.8片, 字扩展4, 位扩展2   D.8片, 字扩展2, 位扩展4
A. 16片, 字扩展8, 位扩展2
解题思路：芯片扩展公式 ① 总片数 = (目标容量)/(单片容量) = (16K×8)/(2K×4) = 8×2 = 16片 ② 字扩展（地址范围扩展）：16K/2K = 8 倍（地址线增加3根） ③ 位扩展（数据宽度扩展）：8/4 = 2 倍
记忆方法
字扩展 → 增加地址线，扩大寻址范围；位扩展 → 增加数据线，加宽数据宽度

二、计算题（21-25）
21将 (21.8125)10 转换成 IEEE754 标准的 32 位浮点数存储格式，写出二进制结果和十六进制结果。
二进制：0 10000011 01011101000000000000000   十六进制：41AE8000H
第1步：将十进制转为二进制 整数部分 21：21 ÷ 2 = 10余1, 10÷2=5余0, 5÷2=2余1, 2÷2=1余0, 1÷2=0余1 → 从下往上 = 10101 小数部分 0.8125：0.8125×2=1.625, 0.625×2=1.25, 0.25×2=0.5, 0.5×2=1.0 → 1101 合并：21.8125 = 10101.1101 第2步：规格化（写成 1.xxxx × 2n 形式） 10101.1101 = 1.01011101 × 24 小数点左移了4位，指数 = 4 第3步：填充 IEEE754 各字段（32位） ① 符号位 S（1位）：正数 → 0 ② 阶码 E（8位）= 指数 + 127（偏移量）= 4 + 127 = 131 131 = 128+2+1 = 10000011 ③ 尾数 M（23位）= 规格化后小数点后的部分，低位补0 01011101 → 01011101000000000000000 第4步：拼接结果 0 | 10000011 | 01011101000000000000000 每4位一组转16进制：0100 0001 1010 1110 1000 0000 0000 0000 = 41AE8000H
举一反三 · 公式
IEEE754 单精度 = (-1)S × 1.M × 2E-127
S=符号位(1位)，E=阶码(8位，移127码)，M=尾数(23位，隐含1.)
22已知 x=+11011，y=-10101，用变形补码计算 x+y，同时指出是否有溢出。
x + y = +00110，无溢出
第1步：写出 x 和 y 的变形补码 (双符号位) x = +11011 → 变形补码 = 00, 011011（双符号位为正） y = -10101，先求 +10101 = 010101 取反 = 101010，+1 = 101011 → 变形补码 = 11, 101011（双符号位为负） 第2步：变形补码相加（逐位相加，进位保留） 00, 011011 + 11, 101011 ──────────── 00, 000110（最高位进位丢弃） 第3步：判断溢出 双符号位 = 00 → 一致 → 无溢出 结果为正数，去掉双符号位 = +00110
举一反三 · 溢出判断规则
双符号位 00 → 正数，无溢出
双符号位 01 → 正溢出（两个正数相加超过正数范围）
双符号位 10 → 负溢出（两个负数相加超过负数范围）
双符号位 11 → 负数，无溢出
23假设有一个具有 24 位地址和 8 位字长的存储器，求：
(1) 该存储器能存储多少字节的信息？
(2) 若存储器由 4M×1 位的 RAM 芯片组成，需要多少片？
(1) 16MB    (2) 32 片
(1) 计算总容量 24位地址 → 224 个存储单元 8位字长 → 每个单元 1 字节 总容量 = 224 × 1B = 16MB（224 = 16M） (2) 计算所需芯片数 目标容量 = 16M × 8位 每片容量 = 4M × 1位 所需片数 = (16M × 8) / (4M × 1) = 4 × 8 = 32 片 其中：字扩展 = 16M/4M = 4，位扩展 = 8/1 = 8
举一反三 · 芯片扩展
字扩展（扩大地址范围）：所需倍数 = 目标存储单元数 / 单芯片存储单元数
位扩展（扩大数据宽度）：所需倍数 = 目标数据位数 / 单芯片数据位数
总片数 = 字扩展倍数 × 位扩展倍数
24CPU 执行程序时，Cache 完成存取 1900 次，主存完成 100 次，Cache 周期 50ns，主存周期 250ns，求 Cache 命中率和平均访问时间。
命中率 = 0.95    平均访问时间 = 60 ns
第1步：计算命中率 命中率 H = Cache命中次数 / 总访问次数 = 1900 / (1900 + 100) = 1900 / 2000 = 0.95 = 95% 第2步：计算平均访问时间（公式 Tavg = H × Tc + (1-H) × Tm） Tavg = 0.95 × 50 + 0.05 × 250 = 47.5 + 12.5 = 60 ns
举一反三 · Cache 系统公式
命中率 H = Cache命中次数 / (Cache命中 + 主存访问)
平均访问时间 Tavg = H × Tc + (1-H) × Tm
性能提升倍数 = Tm / Tavg
注：若主存访问时也访问 Cache（同时访问），用 Tavg = Tc + (1-H) × Tm
25某指令流水线有取指(IF)、译码(ID)、执行(EX)、写回(WB)四个过程，每级 100ns，20 条指令连续输入，计算吞吐率和加速比。
吞吐率 ≈ 8.7×106 条/秒    加速比 ≈ 3.48
第1步：计算流水线总时间 流水线有 k=4 级，执行 n=20 条指令 总时间 = (k + n - 1) × 时钟周期 = (4 + 20 - 1) × 100 = 2300 ns 第2步：计算吞吐率（Throughput） 吞吐率 = 指令数 / 总时间 = 20 / (2300 × 10-9) = 20 / 2.3 × 10-6 ≈ 8.7 × 106 条/秒 第3步：计算加速比（Speedup） 非流水线总时间 = n × k × 时钟周期 = 20 × 4 × 100 = 8000 ns 加速比 = 非流水线时间 / 流水线时间 = 8000 / 2300 ≈ 3.48
举一反三 · 流水线公式
k级流水线，n条指令，时钟周期T：
① 总时间 Ttotal = (k + n - 1) × T
② 吞吐率 TP = n / Ttotal（理想最大吞吐率 = 1/T）
③ 加速比 S = (n × k × T) / ((k + n - 1) × T) ≈ k（n→∞时）
④ 流水线效率 E = (n × k × T) / (k × Ttotal) = n / (k + n - 1) ≈ 1（n → ∞）
